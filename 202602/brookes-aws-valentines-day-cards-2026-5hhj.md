<h2>
  
  
  Compute
</h2>

<p><a href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fin3izv8qidlq4t1j00xb.png" class="article-body-image-wrapper"><img src="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fin3izv8qidlq4t1j00xb.png" alt="Valentine's card with Amazon EC2 icon that reads " width="800" height="800"></a></p>

<p><a href="https://aws.amazon.com/ec2/instance-types/?trk=0fc6058e-ef5a-4fc9-bc07-6efe2c3c9de4&amp;sc_channel=el" rel="noopener noreferrer">Amazon EC2 instance types</a> come in hundreds of configurations with names like c7gn.xlarge, r6i.metal, and x8aedz. They are all very useful but I dread ever having to say them out loud on stage. </p>

<p><a href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2F0tpcr2t068mix4ti7jt3.png" class="article-body-image-wrapper"><img src="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2F0tpcr2t068mix4ti7jt3.png" alt="Valentine's card with EC2 Spot Instances icon that reads " width="800" height="800"></a></p>

<p><a href="https://aws.amazon.com/ec2/spot/?trk=0fc6058e-ef5a-4fc9-bc07-6efe2c3c9de4&amp;sc_channel=el" rel="noopener noreferrer">EC2 Spot Instances</a> give you up to 90% off On-Demand pricing, but AWS can take them back with just two minutes' warning. Like love, they can be fleeting. </p>

<p><a href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fb0w0454jjxp81sd3esjo.png" class="article-body-image-wrapper"><img src="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fb0w0454jjxp81sd3esjo.png" alt="Valentine's card with Amazon EC2 icon that reads " width="800" height="800"></a></p>

<p>In <a href="https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/ec2-security-groups.html?trk=0fc6058e-ef5a-4fc9-bc07-6efe2c3c9de4&amp;sc_channel=el" rel="noopener noreferrer">EC2 security groups</a> , 0.0.0.0/0 means open to the entire internet, which is a rookie error. Closing it down to a single trusted source is the security equivalent of making it Facebook Official.</p>

<p><a href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fdtwmi8uq26parcnrp39b.png" class="article-body-image-wrapper"><img src="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fdtwmi8uq26parcnrp39b.png" alt="Valentine's card with Amazon EC2 icon that reads " width="800" height="800"></a></p>

<p><a href="https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/ec2-key-pairs.html?trk=0fc6058e-ef5a-4fc9-bc07-6efe2c3c9de4&amp;sc_channel=el" rel="noopener noreferrer">EC2 key pairs</a> use SSH keys for secure instance access. This means you need the private key to connect, so you shouldn't be sharing this with anyone.</p>

<p><a href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fv68moazkkr76lc17kxwf.png" class="article-body-image-wrapper"><img src="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fv68moazkkr76lc17kxwf.png" alt="Valentine's card with AWS Graviton5 icon that reads " width="800" height="800"></a></p>

<p><a href="https://aws.amazon.com/ec2/instance-types/m9g/?trk=0fc6058e-ef5a-4fc9-bc07-6efe2c3c9de4&amp;sc_channel=el" rel="noopener noreferrer">AWS Graviton5</a> was launched at re:Invent 2025 and is AWS's most powerful and energy-efficient processor.</p>

<p><a href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fx664t73164rw3alfo67m.png" class="article-body-image-wrapper"><img src="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fx664t73164rw3alfo67m.png" alt="Valentine's card with AWS Trainium3 icon that reads " width="800" height="800"></a></p>

<p><a href="https://aws.amazon.com/ai/machine-learning/trainium/?trk=0fc6058e-ef5a-4fc9-bc07-6efe2c3c9de4&amp;sc_channel=el" rel="noopener noreferrer">AWS Trainium3</a> is custom silicon AWS designed from scratch for AI training. Other times, people use general-purpose chips, but this is much more efficient and it's what your crush deserves for their generative AI workloads.</p>

<p><a href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fzawrbor6zbd5nh54uhhu.png" class="article-body-image-wrapper"><img src="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fzawrbor6zbd5nh54uhhu.png" alt="Valentine's card with AWS Trainium2 icon that reads " width="800" height="800"></a><br>
<a href="https://aws.amazon.com/ai/machine-learning/trainium/?trk=0fc6058e-ef5a-4fc9-bc07-6efe2c3c9de4&amp;sc_channel=el" rel="noopener noreferrer">AWS Trainium2</a> chips are purpose-built for ML training and cost a lot less than GPUs for the same work.</p>

<p><a href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fdgy7c9wcdok0tlcqfnj9.png" class="article-body-image-wrapper"><img src="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fdgy7c9wcdok0tlcqfnj9.png" alt="Valentine's card with AWS Auto Scaling icon that reads " width="800" height="800"></a></p>

<p><a href="https://aws.amazon.com/ec2/autoscaling/?trk=0fc6058e-ef5a-4fc9-bc07-6efe2c3c9de4&amp;sc_channel=el" rel="noopener noreferrer">EC2 Auto Scaling</a> adds and removes compute capacity based on demand so your app never runs out of room to grow, just like your love.</p>

<h2>
  
  
  Serverless
</h2>

<p><a href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fonsa3isdkgk58omc8x44.png" class="article-body-image-wrapper"><img src="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fonsa3isdkgk58omc8x44.png" alt="Valentine's card with AWS Lambda icon that reads " width="800" height="800"></a></p>

<p><a href="https://aws.amazon.com/blogs/compute/understanding-and-remediating-cold-starts-an-aws-lambda-perspective/?trk=0fc6058e-ef5a-4fc9-bc07-6efe2c3c9de4&amp;sc_channel=el" rel="noopener noreferrer">Lambda cold starts</a> happen when a function hasn't been called in a while and needs to get set up before it can get to work - like setting up a kitchen before it's time to cook. The first request can be slow, but then it warms up.</p>

<p><a href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fz5jr54hlera7j9wosyme.png" class="article-body-image-wrapper"><img src="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fz5jr54hlera7j9wosyme.png" alt="Valentine's card with AWS Lambda icon that reads " width="800" height="800"></a></p>

<p><a href="https://aws.amazon.com/lambda/?trk=0fc6058e-ef5a-4fc9-bc07-6efe2c3c9de4&amp;sc_channel=el" rel="noopener noreferrer">Lambda</a> functions have a maximum execution time of 15 minutes and are billed per millisecond.</p>

<p><a href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Folab7s21k75edr2r464u.png" class="article-body-image-wrapper"><img src="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Folab7s21k75edr2r464u.png" alt="Valentine's card with AWS Lambda Durable Functions icon that reads " width="800" height="800"></a></p>

<p><a href="https://docs.aws.amazon.com/lambda/latest/dg/durable-functions.html?trk=0fc6058e-ef5a-4fc9-bc07-6efe2c3c9de4&amp;sc_channel=el" rel="noopener noreferrer">Lambda Durable Functions</a> can pause and wait for up to a year, and you don't pay a thing while they're waiting.</p>

<p><a href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fnrft0zjzpehtz12lom71.png" class="article-body-image-wrapper"><img src="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fnrft0zjzpehtz12lom71.png" alt="Valentine's card with Amazon API Gateway icon that reads " width="800" height="800"></a></p>

<p><a href="https://aws.amazon.com/api-gateway/?trk=0fc6058e-ef5a-4fc9-bc07-6efe2c3c9de4&amp;sc_channel=el" rel="noopener noreferrer">Amazon API Gateway</a> sits in front of your backend, so every single request has to pass through it for authentication, throttling, and routing before anything reaches your services.</p>

<p><a href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fbtffff3y9qc7vq14n7md.png" class="article-body-image-wrapper"><img src="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fbtffff3y9qc7vq14n7md.png" alt="Valentine's card with AWS Step Functions icon that reads " width="800" height="800"></a></p>

<p><a href="https://aws.amazon.com/step-functions/?trk=0fc6058e-ef5a-4fc9-bc07-6efe2c3c9de4&amp;sc_channel=el" rel="noopener noreferrer">AWS Step Functions</a> orchestrates workflows as state machines. So, "state" is both a workflow stage and an emotional condition, and then "step" is the actual technical term for a transition here.</p>

<p><a href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2F7sxo1thbk8pu5i0r9ghx.png" class="article-body-image-wrapper"><img src="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2F7sxo1thbk8pu5i0r9ghx.png" alt="Valentine's card with Amazon ECS icon that reads " width="800" height="800"></a></p>

<p>In <a href="https://aws.amazon.com/ecs/?trk=0fc6058e-ef5a-4fc9-bc07-6efe2c3c9de4&amp;sc_channel=el" rel="noopener noreferrer">Amazon ECS</a> task definitions, the "essential" boolean flag means if that container fails, everything stops.</p>

<p><a href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fh6mm0az54qaclry63u6c.png" class="article-body-image-wrapper"><img src="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fh6mm0az54qaclry63u6c.png" alt="Valentine's card with Amazon EKS icon that reads " width="800" height="800"></a></p>

<p><a href="https://aws.amazon.com/eks/?trk=0fc6058e-ef5a-4fc9-bc07-6efe2c3c9de4&amp;sc_channel=el" rel="noopener noreferrer">Amazon EKS</a> is a fully managed Kubernetes service, so AWS runs the control plane and you don't have to.</p>

<h2>
  
  
  Databases
</h2>

<p><a href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2F29ybbgq8i01ch0wk4j1l.png" class="article-body-image-wrapper"><img src="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2F29ybbgq8i01ch0wk4j1l.png" alt="Valentine's card with Amazon DynamoDB icon that reads " width="800" height="800"></a></p>

<p><a href="https://aws.amazon.com/dynamodb/?trk=0fc6058e-ef5a-4fc9-bc07-6efe2c3c9de4&amp;sc_channel=el" rel="noopener noreferrer">Amazon DynamoDB</a> gives you single-digit millisecond response times at any scale, so this really is true love.</p>

<p><a href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2F65189k9t3duof7at64ww.png" class="article-body-image-wrapper"><img src="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2F65189k9t3duof7at64ww.png" alt="Valentine's card with Amazon Aurora icon that reads " width="800" height="800"></a></p>

<p><a href="https://aws.amazon.com/rds/aurora/?trk=0fc6058e-ef5a-4fc9-bc07-6efe2c3c9de4&amp;sc_channel=el" rel="noopener noreferrer">Aurora</a> Multi-AZ keeps a replica ready to take over as the primary in under 30 seconds if something goes wrong. </p>

<p><a href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fzmbkouzgy1nt3qfgvcs8.png" class="article-body-image-wrapper"><img src="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fzmbkouzgy1nt3qfgvcs8.png" alt="Valentine's card with Amazon RDS icon that reads " width="800" height="800"></a></p>

<p><a href="https://aws.amazon.com/rds/?trk=0fc6058e-ef5a-4fc9-bc07-6efe2c3c9de4&amp;sc_channel=el" rel="noopener noreferrer">RDS</a> Multi-AZ keeps a standby copy of your database in a different Availability Zone, always in sync, always ready and waiting.</p>

<p><a href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fkasx5ap9det9sh535zwd.png" class="article-body-image-wrapper"><img src="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fkasx5ap9det9sh535zwd.png" alt="Valentine's card with Aurora DSQL icon that reads " width="800" height="800"></a></p>

<p><a href="https://aws.amazon.com/rds/aurora/dsql/?trk=0fc6058e-ef5a-4fc9-bc07-6efe2c3c9de4&amp;sc_channel=el" rel="noopener noreferrer">Aurora DSQL</a> is a serverless SQL database that patches, scales, and manages itself, so you can stop babysitting infrastructure and spend your time planning dates.</p>

<p><a href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fpieaefarxhxn3nz7r0yb.png" class="article-body-image-wrapper"><img src="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fpieaefarxhxn3nz7r0yb.png" alt="Valentine's card with Amazon Neptune icon that reads " width="800" height="800"></a></p>

<p><a href="https://aws.amazon.com/neptune/?trk=0fc6058e-ef5a-4fc9-bc07-6efe2c3c9de4&amp;sc_channel=el" rel="noopener noreferrer">Neptune</a> is a graph database so data is stored as nodes connected by edges, and edges are literally called "relationships." </p>

<p><a href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fmkqdq2osonapxhy7hfja.png" class="article-body-image-wrapper"><img src="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fmkqdq2osonapxhy7hfja.png" alt="Valentine's card with Amazon ElastiCache icon that reads " width="800" height="800"></a></p>

<p><a href="https://aws.amazon.com/elasticache/?trk=0fc6058e-ef5a-4fc9-bc07-6efe2c3c9de4&amp;sc_channel=el" rel="noopener noreferrer">ElastiCache</a> keeps your most-accessed data in memory so you get it back in microseconds, without repeat trips to the database.</p>

<p><a href="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fec54nh2zt2z8jw6ap65y.png" class="article-body-image-wrapper"><img src="https://media2.dev.to/dynamic/image/width=800%2Cheight=%2Cfit=scale-down%2Cgravity=auto%2Cformat=auto/https%3A%2F%2Fdev-to-uploads.s3.amazonaws.com%2Fuploads%2Farticles%2Fec54nh2zt2z8jw6ap65y.png" alt="Valentine's card with Valkey logo that reads " width="800" height="800"></a></p>

<p><a href="https://aws.amazon.com/elasticache/what-is-valkey/?trk=0fc6058e-ef5a-4fc9-bc07-6efe2c3c9de4&amp;sc_channel=el" rel="noopener noreferrer">Valkey</a> is open-source, so this one is not an AWS card, but <a href="https://www.linkedin.com/in/madelyn-olson-valkey/" rel="noopener noreferrer">Madelyn</a> asked for this card last year and it was one of my favourites, so it's back for 2026! Valkey is an in-memory key-value store supported by ElastiCache and MemoryDB, and it's the most permissive open source alternative to Redis. </p>
