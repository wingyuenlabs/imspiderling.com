<p>With StreamKit.dev one can build and run real-time media pipelines on your own infrastructure. Speech-to-text, voice agents, live audio processing — composable, observable, self-hosted. It is totally open source. Full references and description at <a href="https://streamkit.dev/" rel="noopener noreferrer">https://streamkit.dev/</a></p>

<p>Who is this for?<br>
StreamKit is built for developers who need to process real-time media — whether you’re building voice features for an app, prototyping an AI audio pipeline, or self-hosting alternatives to cloud speech APIs.</p>

<p>What you can build<br>
Live transcription — Ingest audio via MoQ, run Whisper or SenseVoice STT, stream transcription updates to clients<br>
Voice agents — TTS-powered bots using Kokoro, Piper, or Matcha that respond to audio input<br>
Real-time translation — Bilingual streams with live subtitles using NLLB or Helsinki models<br>
Audio processing — Mixing, gain control, format conversion, encoding/decoding pipelines<br>
Content analysis — VAD for speech detection, keyword spotting, or custom safety filters.</p>

<p>one can try the powerful engine at <a href="https://demo.streamkit.dev/" rel="noopener noreferrer">https://demo.streamkit.dev/</a> </p>
