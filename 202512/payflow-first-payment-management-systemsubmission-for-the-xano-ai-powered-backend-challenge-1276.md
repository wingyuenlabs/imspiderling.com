<h2>
  
  
  What I Built
</h2>

<p>PayFlow is a modern, AI-first payment management system that allows users to:</p>

<p>Create accounts and authenticate securely</p>

<p>Manage digital wallets</p>

<p>Top up balances using real Stripe payments (test mode)</p>

<p>Send money to other users instantly</p>

<p>Track transactions with a full audit trail</p>

<p>Operate in a multi-account/team environment</p>

<p>The problem PayFlow solves is building a scalable, production-ready payment backend quickly, while still applying human engineering judgment on security, data modeling, and workflows.</p>

<p>Instead of relying purely on AI output, I used XanoScript to generate an initial backend, then refined and hardened it inside Xano to meet real-world payment and security expectations.</p>

<h2>
  
  
  Challenges &amp; Learnings
</h2>

<p>While working with the XanoScript extension inside VS Code, I learned that backend generation is not a live-sync process.</p>

<p>The AI-generated output serves as a starting blueprint, but API groups and endpoint wiring must be finalized inside the Xano dashboard. I had to manually create API groups and attach refined logic to ensure correct authentication, validation, and Stripe handling.</p>

<p>Additionally, the API base URL changed during iteration due to Xanoâ€™s environment-based deployment and API versioning model. This required updating frontend environment variables during testing, reinforcing the importance of environment management in real production systems.</p>

<p>This process highlighted the distinction between AI-assisted generation and human-led system refinement, which ultimately resulted in a more secure and maintainable backend.</p>
