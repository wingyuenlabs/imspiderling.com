<h2>
  
  
  ‚öôÔ∏è Technical Highlights
</h2>

<p>We wanted to learn as much and take as much profit as possible from Go ecosystem, to adopt the same stack of some tools we love - and admire - to use on a daily basis : </p>

<ul>
<li>Language : <code>Golang</code> </li>
<li>Delivery : <code>brew</code> to deliver/install efficiently as fast - and yet secure - as we want</li>
<li>Release Management : <a href="https://github.com/goreleaser/goreleaser" rel="noopener noreferrer"><code>goreleaser/goreleaser</code></a>
</li>
<li>
<a href="https://github.com/spf13/cobra" rel="noopener noreferrer"><code>spf13/cobra</code></a> for CLI</li>
<li>
<a href="https://github.com/charmbracelet/fang" rel="noopener noreferrer"><code>charmbracelet/fang</code></a> on top of Cobra to give glamour to our cli</li>
<li>
<a href="https://github.com/charmbracelet/lipgloss" rel="noopener noreferrer"><code>charmbracelet/lipgloss</code></a> for styish terminal outputs : colors, arrays, links, <code>markdown</code> display and redirection t files</li>
<li>
<code>duckdb</code> as primary export format</li>
<li>
<a href="https://osv.dev/#use-vulnerability-scanner" rel="noopener noreferrer"><code>osv.dev</code></a> to continuously check for vulnerabilities in our base code</li>
<li>
<a href="https://codeql.github.com/" rel="noopener noreferrer">Code QL</a> as part of our CI to ensure our tool does not bring security issues</li>
<li>
<a href="https://docusaurus.io/" rel="noopener noreferrer">Docusaurus</a> for nice professional website (work in progress)</li>
<li>
<a href="https://github.com/charmbracelet/crush" rel="noopener noreferrer"><code>crush</code></a> and <a href="https://github.com/google-gemini/gemini-cli" rel="noopener noreferrer"><code>gemini-cli</code></a> to evaluate the design of our cli</li>
<li>
<code>pandoc</code> for document post-processing (eg. transform output <code>markdown</code> into <code>html</code> or <code>pdf</code>...)</li>
<li>
<code>LaTeX</code> to build the <a href="https://x.com/rastadidi/status/1995616096351891548" rel="noopener noreferrer">cheetsheat</a>
</li>
<li>The way we an put together <code>geol</code> and <code>trivy</code> (with <code>crush</code> or <code>gemini-cli</code>) at work together to build efficient and nice looking technical debt and security reports</li>
<li>
<a href="https://distrobox.it/" rel="noopener noreferrer"><code>distrobox</code></a> to test install and binaries on various OS and architectures</li>
<li>Now, I'm starting to focus on what can be done around <code>geol</code> outputs to automate reporting, with a professional data-stack, like <a href="https://rmarkdown.rstudio.com/" rel="noopener noreferrer"><code>Rmarkdown</code></a> or <a href="https://quarto.org/" rel="noopener noreferrer"><code>quarto</code></a> to make professional looking technical debt reports</li>
<li>Also, I'm thinking about the most efficient way to share EOL's datas into other systems to make technical debt easier to monitor</li>
</ul>
